# 프레임워크
개발할 때 기본 구조를 이미 만들어 놓은 틀이다.
## 비유
### 프레임 워크가 없을 때

맨땅에서 집을 짓는다 하면

-설계도를 직접 만들고 

-벽도 직접 세우고

-전기, 수도 다 직접

이렇게 하면 시간이 오래 걸리고, 실수가 많다.
### 프레임 워크가 있을때
이미 만들어진 아파트

-구조가 있음

-규칙이 있음

-바로 꾸미기만 하면 됨

이렇게 프레임워크를 사용하면 빠르고 안정적이다.

## 프레임 워크의 특징
### 1. 뼈대가 제공된다

  기본구조를 만들어줌

 -로그인 처리 구조

 -서버 구조

 -DB 연결 구조
### 2. 규칙이 있다

  이렇게 코딩해! 라고 강제함

  -파일 위치가 정해짐

  -클래스의 역할이 정해짐
### 3. 제어의 역전

프로그램 흐름을 내가 아니라 프레임워크가 관리

->이것을 loC 라고 한다.
